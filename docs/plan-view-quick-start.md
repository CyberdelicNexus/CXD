# Plan View Quick Start Guide

## Accessing Plan View
Click the **Plan** button in the navbar (between Hypercube and Canvas)

## Making Cards Actionable

### Method 1: Markdown Checkboxes
Add task checkboxes in your card content:
```
- [ ] Design wireframes
- [ ] Build prototype
- [x] Define requirements
```

### Method 2: Mark as Task
1. Select a card in canvas
2. Click the **checkmark icon** (☑️) in the context menu toolbar
3. Purple dot appears on card corner

### Method 3: Hypercube Tags
Tag a card with any hypercube face:
- Reality Planes
- Sensory Domains  
- Presence Types
- State Mapping
- Trait Mapping
- Meaning Architecture

## Plan View Features

### Views
- **Kanban**: Drag cards between status columns
- **Table**: Sort and filter with detailed columns
- **Timeline**: Gantt chart with date ranges
- **Calendar**: Monthly view with daily task lists

### Task Properties (editable in Plan view)
- Status (To Do, In Progress, Blocked, Done)
- Priority (Low, Medium, High, Urgent)
- Due Date
- Start Date
- Assignee
- Estimated Hours

### Filters
- Show/hide completed tasks
- Filter by status, priority, or hypercube face
- Filter by task type (markdown, actionable, tagged)

## Navigation

### From Plan to Canvas
Click the arrow button (→) on any task card to jump to its location in the canvas

### From Canvas to Plan
Click the Plan button in navbar - all actionable cards appear automatically

## Tips
- Cards keep their styling and content - task metadata is non-visual
- Checkbox progress automatically updates completion percentage
- Use hypercube tags to organize tasks by experience dimension
- All changes sync instantly between Plan and Canvas views
